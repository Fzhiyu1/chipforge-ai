# 01 - 知识图谱数据结构设计

## 探索目标

设计适合 FPGA/Verilog 开发的知识图谱数据结构。

## 待讨论话题

### 1. 节点类型设计

需要支持哪些类型的知识节点？

- 语法节点（关键字、语句、表达式）
- 模块节点（module、interface）
- 设计模式节点
- 最佳实践节点
- 常见错误节点

### 2. 关系类型设计

节点之间有哪些关系？

- `包含` - 模块包含端口
- `引用` - 代码引用模块
- `相关` - 概念相关联
- `解决` - 模式解决问题

### 3. 存储方案

- 图数据库（Neo4j）
- 关系数据库 + JSON
- 内存图结构
- 文件存储（JSON/YAML）

## 评估标准

- 查询效率
- 扩展性
- 维护成本
- 与 MCP 集成难度
